== 📝 Project Overview

This is a professional-grade full-stack application. It features a robust ASP.NET Core 9.0 backend for managing employee records, implementing a manual HATEOAS pattern. The frontend is built with Angular 18, providing a modern and reactive user interface.

== 🚀 Key Features

Full CRUD Operations: Create, Read, Update, and Delete employees.

Manual HATEOAS Implementation: Every response includes navigational links for a self-documenting API.

Unit Testing: Robust backend logic validated with xUnit and Moq.

Containerization: Fully Dockerized environment with automated testing during the build process.

Clean Architecture: Repository Pattern, DTO mapping with AutoMapper, and decoupled structure.

== 🐳 Docker Support

The project is fully containerized, allowing you to run the entire stack with a single command.

=== Multi-stage Builds

Backend: The Dockerfile uses a multi-stage process. It compiles the code and automatically runs xUnit tests. If any test fails, the Docker image build will stop, ensuring only stable code is deployed.

Frontend: The Angular app is compiled into production-ready static files and served via a high-performance Nginx container.

=== Running with Docker Compose
Ensure you have Docker and Docker Compose installed, then run:

[source,bash]

docker-compose up --build

Once the containers are up:

Frontend: Access at http://localhost:4200

BackEnd access and Swagger UI: Access at http://localhost:5265/index.html

== 🚦 Getting Started (Local Development)

=== Prerequisites

.NET 9 SDK

Node.js (Latest LTS) & Angular CLI

=== Installation
. Clone the repository:
[source,bash]
git clone https://github.com/dev-clement/EmployeeManagement.git

. Run the Backend:
[source,bash]
dotnet run --project EmployeeManagement

. Run the Frontend:
[source,bash]
cd frontend && npm install && ng serve

== 🛡 Architecture & Quality

HATEOAS: The API implements hypermedia links (self, update_employee, delete_employee) to guide the client through the application state.

Safety: AutoMapper is configured to ignore Id overwrites during updates to prevent primary key corruption.

Isolation: The backend and frontend communicate via a strict CORS policy defined in Program.cs.
